# Tasted — backend

Photograph a foreign menu → get dish descriptions, allergen probabilities and dietary info.

## Stack

- **FastAPI** (async) — API layer
- **SQLAlchemy 2.0 async + asyncpg** — DB access
- **Postgres + pgvector** — relational data *and* the semantic dish cache
- **Celery + Redis** — background menu processing (extract -> cache -> enrich)
- **uv** — dependency & environment management (`pyproject.toml` + `uv.lock`)
- **Docker Compose** — api + worker + db + redis

## Run with Docker (recommended)

```sh
docker compose up --build
```

- API:      http://localhost:8000
- Docs:     http://localhost:8000/docs
- Health:   http://localhost:8000/health
- DB check: http://localhost:8000/health/db

The API container mounts `./app`, so code changes reload live.

## Run locally (without Docker)

Requires a Postgres with the `pgvector` extension reachable at the
`POSTGRES_*` connection details in `.env` (use `POSTGRES_HOST=localhost`).

```sh
cp .env.example .env
uv sync
uv run uvicorn app.main:app --reload
```

Connection settings are split into components (`POSTGRES_USER`, `POSTGRES_PASSWORD`,
`POSTGRES_HOST`, `POSTGRES_PORT`, `POSTGRES_DB`); `app.config.Settings` assembles them
into `settings.database_url`.

## Database migrations (Alembic)

The schema is owned by Alembic (async). The API container runs `alembic upgrade head`
before starting the server, so migrations apply automatically on `docker compose up`.

```sh
# create a migration after changing app/models/*
uv run alembic revision --autogenerate -m "add x"
uv run alembic upgrade head        # apply
uv run alembic downgrade -1        # roll back one
uv run alembic current             # show applied revision
```

The DB URL comes from `app.config.settings.database_url` (assembled from the `POSTGRES_*`
env vars) via `alembic/env.py` — not hardcoded in `alembic.ini`. New autogenerated migrations that use the `VECTOR` type
need `import pgvector` at the top of the file (Alembic doesn't add it automatically).

  llm/               # LLM boundary

    base.py          #   Base, EMBEDDING_DIM

    user.py          #   User (Clerk identity + synced prefs blob)

    attribute.py     #   DishAttribute (0-100 scores), DishVote (arrow nudges)
    currency.py      #   Currency (daily rate per 1 EUR)
  domain/            # Pydantic domain models — what services speak in
    dish.py          #   DishInfo, Allergen, DietaryFlag
    menu.py          #   Money, ExtractedMenuItem, MenuExtraction
    preferences.py   #   Preferences, WatchChip (mirror of User.prefs)
  repositories/      # DB access — wrap the session, return ORM

    dishes.py        #   DishRepository (get/list/add/find_nearest)
    menus.py         #   MenuRepository (get/list_for_user)
    scans.py         #   ScanRepository (worker-side scan/item writes)
    users.py         #   UserRepository (get/by_clerk_id/insert-ignore)
    currencies.py    #   CurrencyRepository (list/get/upsert_rates)
  schemas/           # Pydantic API request/response — the wire shapes
    dish.py          #   PhotoOut, DishOut (wraps domain DishInfo)
    menu.py          #   MenuOut, MenuItemOut, MenuSummaryOut, statuses
    currency.py      #   CurrencyOut

## External integrations (all optional in local dev)

Each has a no-credential fallback (same interface, swapped implementation) so
the whole stack demos offline:

- **Clerk** (`CLERK_JWKS_URL`) — real mode verifies session JWTs against the
  JWKS and maps the `sub` claim to a user (load-or-create). **Unset ⇒ faked
  but ready**: the same dependency trusts the token without verifying it — a
  real Clerk JWT is decoded unverified, a plain string like `alice` is used as
  the identity, and no header ⇒ a fixed `dev_user`. So the app's real sign-in
  flow drives per-user behavior in the demo, and curl just needs
  `-H "Authorization: Bearer alice"`. See `app/auth.py`.
- **S3 / R2 / MinIO** (`S3_BUCKET`, `AWS_*`, `AWS_ENDPOINT_URL`) — menu photos
  and user dish photos. **Unset ⇒ `LocalStorage`**: same `put/get/public_url`
  API, writes under `uploads/` (bind-mounted into the worker), and the files
  are served at `/uploads/...` so `public_url` resolves in the demo. See
  `app/services/storage.py`.
- **OpenAI** (`OPENAI_API_KEY`) — ChatGPT vision extraction, per-dish
  enrichment, and embeddings for the cache. Unset ⇒ deterministic stub AI.
